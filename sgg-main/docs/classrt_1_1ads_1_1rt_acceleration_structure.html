Detailed Description

template<class T = Eigen::Vector3f>
class rt::ads::rtAccelerationStructure< T >

Prototype for all acceleration structures (AS) used in the framework. Provides nearest data and distance queries for the represented geometry.

A (spatial) acceleration structure provides fast methods to query distances and closest elements to a particular input element based on a user-defined metric and data the data dimensions. Typically, they store positions in space along with some additional data (a point "payload"). However, for our framework we do not necessarily assume that the indexing is performed in 3 dimensions, although this may be the usual case. For example, a particular AS may require that color is a very important point "proximity" criterion and must be included in the dimensionality of the queried data, resulting in a 6D data structure.

Certain data structures that are implemented in the framework assume a dimensionality of 3 (e.g. hierarchical grids), while others are not required to do so and can therefore operate in higher dimensions (e.g. linear structures, distance fields)

All AS must inherit from rtAccelerationStructure in order to have a unified interface for querying data from them. Typical 3D acceleration structures can be inherited and built from this class prototype such as kd trees, distance fields and n-level hierarchical grids. To support geometry that may have more that AS associated with it (some algorithms require this), use the derived class rtMultiAccelerationStructure.

Think of the acceleration structure as a container of entities that are "associated" with the input elements of type T but are not necessarily the elements themselves. For example, one can implement a distance field as a derived class of rtAccelerationStructure (see rtDistanceField). Although the template type T may be a multidimensional entity, not all element coordinates may be used in the construction and query of the AS, the first dictated by the AS type and the latter specified by the distance metric. It is important to note that the template type T must provide the operator [] to access the coordinates of the elements inserted or queried.

rtAccelerationStructure-derived classes need to provide implementations to the build(), addData(), nearest() and distanceToNearest() members. nearest() returns a result code since it is not guaranteed to be a valid operation for some acceleration structures (e.g. distance field). Also, the base class provides an iterator inner class declaration and a begin() and end() methods to obtain a forward input (read-obly) iterator on the internal data. Developers should override both in order to return a meaningful and functional iterator for their derived rtAccelerationStructure class.

Returns the distance to the nearest element to the query input. It is the multiple input version of the distanceToNearest() method.

The method searches and returns the distance of all query points to their respective closest element, according to the distance metric provided in the AS. The distance function provided may or may not use the entire set of coordinates. Whenever possible, use this function instead of the simple distanceToNearest() method, as it favors efficient and parallel queries, depending on implementation. The distance function provided may or may not use the entire set of coordinates and features in the template type T.

Parameters
resultis a pointer to an array variable to contain the query results, if the operation is supported. The result array must be pre-allocated with sufficient space to hold num returned elements.
queryis an array of input elements of type T for which the closest elements are returned, if the operation is supported.
numis the number of points to query.
Returns
an rtResult object with the code set to RT_OK if the operation proceeded notmally and the referenced data in the acceleration structure contained at least one element. If the acceleration structure does not support the operation, the code returned is RT_INVALID_OPERATION. The method returns RT_FAILURE if the data structure is empty and RT_INVALID_OPERATION if it is uninitialized. The message member of the rtResult specializes the reason of the invalid operation.

template<class T >
rtResult rt::ads::rtAccelerationStructure< T >::setDistanceMetric ( std::function< float(const T &, const T &)>  metric)

Sets (overrides) the default distance metric defined for each one of the implementations of the rtAccelerationStructure class prototype.

Initially, each implementation of an acceleration structure defines a default distance operator, typically related to Euclidean distance in 3D cartesian coordinates. The distance metric is used for comparing and locating objects within the AS. You can overide the metric in order to impose a user-defined function for this task. For example, you may want to include other attributes of the stored element in the metric, such as a normal directin or color. It is invalid to assign a distance metric to an already initialized structure.

Returns
an rtResult object with the code set to RT_OK if the metric was successfully set. If the data structure is already initialized (via the build()) function, the code field of the rtResult is set to RT_INVALID_OPERATION. No other code is applicable.
Parameters
metricis the function object that corresponds to the global function, functor or closure to execute for the distance metric.
